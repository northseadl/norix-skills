# Explore 模式 Playbook

## 适用时机

1. 用户目标不明确
2. 需要多方向方案探索
3. 需要比较不同风格/交互策略

## 执行步骤

1. 抽取目标与约束（先读 `design-goal-spec-template`）
2. 生成 2-3 个备选方案
3. 每个方案都要写明：
   - 适配场景
   - 核心交互假设
   - 主要风险
4. 不得给出发布结论

## 输出格式

1. 主方向候选
2. 备选方向列表
3. 推荐进入 Production 的候选及理由

## 质量底线（即便 Explore 也必须满足）

1. 不得违反可访问性基本原则
2. 不得给出明显不可实施方案
3. 不得省略关键状态流
