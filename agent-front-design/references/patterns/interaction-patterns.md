# 交互模式库

## 模式选型原则

选择交互模式的依据是**操作风险等级**和**认知复杂度**，而非视觉偏好。

| 风险/复杂度 | 低认知 | 高认知 |
|-----------|-------|-------|
| **低风险** | Direct Manipulation | Inline Edit |
| **高风险** | Command-Confirm | Draft-Review-Publish |

## 模式 A：Direct Manipulation（直接操作）

适用：低风险、即时生效的操作。

流程：`操作 → 即时反馈 → 完成`

示例：
- 切换开关（Toggle）
- 拖拽排序
- 收藏/取消收藏
- 数量增减

设计要求：
- 操作必须有即时视觉反馈（状态变化 < 100ms 感知）
- 支持撤销（Undo）作为安全网
- 网络请求异步处理，乐观更新

## 模式 B：Inline Edit（就地编辑）

适用：轻量属性修改，无需离开当前上下文。

流程：`点击/双击编辑区域 → 就地编辑 → 即时校验 → 确认/取消`

示例：
- 表格单元格编辑
- 标题/名称重命名
- 标签/分类修改

设计要求：
- 编辑态与展示态视觉区分明确（border/背景变化）
- Enter 确认、Escape 取消
- 编辑中若失焦，自动保存或提示
- 校验失败时不允许确认，但保留输入内容

## 模式 C：Command-Confirm（命令确认）

适用：高风险但认知简单的操作。

流程：`触发 → 二次确认（含影响说明）→ 执行 → 结果反馈`

示例：
- 删除操作
- 发布/上线
- 批量操作
- 权限变更

设计要求：
- 确认对话框明确说明**影响范围**（"将删除 3 个文件及其关联数据"）
- 破坏性操作使用红色语义 + 需要输入确认文字
- 操作执行中显示 loading 状态
- 支持操作结果通知（成功/失败/部分成功）

## 模式 D：Draft-Review-Publish（草稿-审核-发布）

适用：高风险且认知复杂的操作。

流程：`创建草稿 → 编辑 → 预览 → 审核 → 发布`

示例：
- 内容创建与发布
- 配置变更与生效
- 模板设计与应用

设计要求：
- 清晰的生命周期状态指示（草稿/待审核/已发布）
- 预览环境尽可能接近生产环境
- 支持版本对比（diff view）
- 发布后支持回滚

## 模式 E：Progressive Disclosure（渐进揭示）

适用：信息量大但用户关注点不同的场景。

流程：`展示核心信息 → 按需展开详情 → 深度探索`

示例：
- 复杂表单的"高级选项"
- 数据面板的下钻
- 错误详情的展开

设计要求：
- 默认只展示 80% 用户需要的信息
- 扩展入口明确但不喧宾夺主
- 展开行为保持页面稳定（不导致内容跳动）
- 记住用户的展开偏好

## 统一规则

1. **反馈时延可感知**：每个用户操作必须在 100ms 内有视觉反馈
2. **错误可恢复**：失败状态必须提供下一步行动指引
3. **不允许静默失败**：后台操作失败时必须通知用户
4. **一致性**：同类操作在所有页面使用相同交互模式
