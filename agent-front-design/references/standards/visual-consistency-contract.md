# 视觉一致性契约

## 目标

保证不同任务与不同设计方向下，视觉系统保持一致性与可维护性。

## 强约束

1. 必须定义基础 token：色彩、间距、字体层级、圆角、边框、阴影
2. 同一语义组件不得出现多套冲突样式
3. 不得用“临时样式”替代系统 token

## 组件语义规则

1. Primary Action：每屏最多一个主行动按钮
2. Secondary Action：与主行动视觉层级明确区分
3. Feedback Component：成功/警告/错误语义区分清晰

## 禁止项

1. 随机色板拼接
2. 无语义的字体尺寸跳变
3. 仅靠颜色表达状态（需配合文本/图标）
4. 未定义 token 即直接写死样式

## 交付检查

1. 提供 token 清单
2. 提供关键组件映射表（语义 -> 组件）
3. 提供视觉偏差风险说明
