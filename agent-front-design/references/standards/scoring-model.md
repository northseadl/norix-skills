# 评分与阻断模型（Production）

## 评分规则

1. 七个维度每维 0-4 分
2. 总分折算到 100：`sum(dimension_score) / 28 * 100`

分值语义：

- `0`：缺失或不可用
- `1`：存在重大缺陷
- `2`：基础可用但有明显风险
- `3`：质量稳定可交付
- `4`：优秀且可复用

## 通过阈值

1. 总分 >= 85
2. 任一维度不得低于 2

## 阻断规则

1. 存在 `P0` 或 `P1` -> 强制阻断发布
2. 任一维度 < 2 -> 强制阻断发布

## 严重级别

1. `P0`: 核心流程不可用 / 明确安全高危
2. `P1`: 主路径严重阻断 / 高风险性能或可访问性问题
3. `P2`: 有替代路径但体验显著下降
4. `P3`: 局部可优化问题

## 输出要求

Production 报告必须包含：

1. 每维分数与理由
2. 总分
3. 阻断项明细
4. 风险缓解计划
5. 发布结论（可发布 / 条件发布 / 不可发布）
