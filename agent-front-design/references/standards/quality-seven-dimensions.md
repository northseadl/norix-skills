# 七维质量定义

用于约束“优秀前端”必须覆盖的七个维度。任何维度缺失都属于不完整交付。

## 1. 目标一致性

检查点：

1. 业务目标与用户目标是否映射到具体页面/流程
2. 范围边界是否明确（in-scope / out-of-scope）
3. 成功指标是否可测量

## 2. 信息架构完整性

检查点：

1. 页面层级是否清晰
2. 导航路径是否闭环
3. 入口策略是否一致

## 3. 交互完整性

检查点：

1. 主流程可达
2. 异常路径完整
3. 状态覆盖完整：`loading/empty/error/success/permission`

## 4. 视觉系统一致性

检查点：

1. token 使用一致（色彩、间距、圆角、阴影）
2. 排版层级一致
3. 组件语义一致（主次按钮、提示、反馈）

## 5. 工程可实施性

检查点：

1. 组件树可落地
2. 状态边界清晰
3. 数据契约明确（来源、结构、失败处理）

## 6. 可访问性与性能

检查点：

1. 语义结构、焦点可见、键盘可达
2. 可读性与对比度达标
3. 性能预算可执行（LCP/TBT/CLS 或等价指标）

## 7. 安全与可观测性

检查点：

1. 输入边界与前端注入风险控制
2. 权限与敏感信息暴露风险识别
3. 关键交互事件具备埋点与告警策略
