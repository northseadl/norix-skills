# 设计原则

## 核心哲学

优秀的前端设计不是装饰，是信息传达的精密工程。每个视觉决策都服务于用户的认知效率和情感体验。

## 六项设计原则

### 1. Intentionality（意图性）

每个设计决策必须可追溯到业务目标或用户需求。

反模式：
- "加个渐变看起来更高级" — 没有理由的装饰
- "用圆角因为流行" — 跟随趋势而非解决问题

正确做法：
- "使用 8px 圆角因为与品牌的友好调性一致"
- "使用线性渐变引导用户视线到 CTA 区域"

### 2. Restraint（克制）

克制是品位的表现。能用留白解决的不加元素，能用一种动效解决的不叠三种。

关键约束：
- 每屏最多 1 个视觉焦点
- 动效时长不超过 400ms（导航过渡除外）
- 色板控制在 1 主色 + 1 辅色 + 灰阶 + 语义色
- 字体族不超过 2 个

### 3. Authenticity（真实性）

设计应该传达品牌独特的声音，而不是 AI 训练数据的统计平均。

真实感来源：
- 源自品牌 DNA 的色彩（不是"看起来科技"的蓝紫）
- 符合内容调性的字体选择（不是默认 Inter）
- 反映业务场景的插画/图标风格（不是抽象 3D 人物）
- 微妙的不完美（有温度的圆角变化、手绘风图标）

### 4. Hierarchy（层次）

信息层次是用户认知效率的基础。清晰的视觉层次让用户不用"读"就能"看懂"。

层次工具（按优先级）：
1. 尺寸对比 — 最强的层次信号
2. 色彩权重 — 通过饱和度和明度区分主次
3. 空间分组 — 相关元素靠近，无关元素远离
4. 排版层级 — 标题/正文/辅助文字的韵律

### 5. Coherence（连贯）

系统级一致性高于单页视觉冲击。用户在多页面间的认知连续性比单屏惊艳更重要。

连贯性检查：
- 同语义的组件在所有页面表现一致
- 操作反馈模式在全应用统一
- 间距和排版节奏在页面间无突变
- Token 驱动的主题切换不破坏信息层次

### 6. Resilience（韧性）

设计必须在真实条件下工作 — 超长文本、空数据、慢网络、小屏幕。

韧性检查清单：
- 文案溢出：超长标题/名称的截断策略
- 极端数据：0 条/1 条/10000 条的渲染策略
- 网络异常：离线/慢速下的骨架屏与错误恢复
- 设备跨度：从 320px 到 2560px 的适配策略

## 设计决策框架

做设计决策时，按以下优先级排序：

1. **用户认知负荷** — 这个决策会增加还是减少用户的思考量？
2. **信息传达效率** — 用户能否在 3 秒内理解页面主旨？
3. **操作成本** — 完成核心任务的步骤数能否更少？
4. **情感共鸣** — 是否传达了品牌期望的情感调性？
5. **工程可维护性** — 是否可以被 Token 系统化管理？
