# 质量门禁

## 七维质量定义

任何维度缺失都属于不完整交付。

### 1. 目标一致性

检查点：
1. 业务目标与用户目标是否映射到具体页面/流程
2. 范围边界是否明确（in-scope / out-of-scope）
3. 成功指标是否可测量（具体数字或可观察行为）

### 2. 信息架构完整性

检查点：
1. 页面层级是否清晰（不超过 3 层嵌套）
2. 导航路径是否闭环（任何页面可回到主路径）
3. 入口策略是否一致（同类内容通过同类入口到达）

### 3. 交互完整性

检查点：
1. 主流程可达（核心任务 ≤ 5 步完成）
2. 异常路径完整（参考状态覆盖规范）
3. 状态覆盖完整：`loading / empty / error / success / permission / partial`

### 4. 视觉系统一致性

检查点：
1. Token 使用一致（色彩、间距、圆角、阴影均引用 Token）
2. 排版层级一致（标题/正文/辅助文字有明确的尺寸节奏）
3. 组件语义一致（主次按钮、提示、反馈在全应用统一）
4. **AI 审美偏离度合格**（参考 anti-ai-aesthetic.md 自检清单）

### 5. 工程可实施性

检查点：
1. 组件树可落地（组件边界清晰、职责单一）
2. 状态边界清晰（什么状态在哪个层级管理）
3. 数据契约明确（来源、结构、失败处理）
4. Token 可实现（CSS Custom Properties 或等效机制）

### 6. 可访问性与性能

检查点：
1. 语义结构正确（HTML5 语义元素、ARIA 标签）
2. 焦点可见、键盘可达（Tab 顺序合理）
3. 色彩对比度达标（WCAG 2.2 AA：正文 4.5:1、大文字 3:1）
4. 性能预算可执行（LCP ≤ 2.5s / INP ≤ 200ms / CLS ≤ 0.1）
5. 动效尊重 `prefers-reduced-motion`

### 7. 安全与可观测性

检查点：
1. 输入边界与前端注入风险控制（XSS 防护）
2. 权限与敏感信息暴露风险识别（不在前端存储 token 明文）
3. 关键交互事件具备埋点（页面曝光、按钮点击、流程完成）

## 评分规则

| 分值 | 语义 |
|------|------|
| 0 | 缺失或不可用 |
| 1 | 存在重大缺陷 |
| 2 | 基础可用但有明显风险 |
| 3 | 质量稳定可交付 |
| 4 | 优秀且可复用 |

总分折算：`sum(dimension_score) / 28 × 100`

## 通过阈值

1. 总分 ≥ 85
2. 任一维度不得低于 2

## 阻断规则

1. 存在 P0 或 P1 → 强制阻断发布
2. 任一维度 < 2 → 强制阻断发布

## 严重级别

| 级别 | 定义 |
|------|------|
| P0 | 核心流程不可用 / 明确安全高危 |
| P1 | 主路径严重阻断 / 高风险性能或可访问性问题 |
| P2 | 有替代路径但体验显著下降 |
| P3 | 局部可优化问题 |

## Production 报告输出要求

1. 每维分数与理由
2. 总分
3. 阻断项明细（P0/P1/P2/P3）
4. 风险缓解计划
5. 发布结论（可发布 / 条件发布 / 不可发布）
