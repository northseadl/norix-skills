# 交付物模板

方案交付按阶段映射到以下模板。每个模板标注了必填（■）和选填（□）字段。

---

## 1. DesignGoalSpec（目标定义）

```markdown
# 设计目标规格

## 业务目标
■ 核心业务诉求（1-3 条）
■ 预期业务指标变化

## 用户目标
■ 目标用户画像
■ 用户核心任务（Jobs to Be Done）

## 范围边界
■ In-scope: [明确包含的功能/页面]
■ Out-of-scope: [明确排除的功能/页面]

## 约束条件
■ 时间约束
■ 技术约束（框架/兼容性/性能）
□ 合规约束
□ 团队约束

## 成功指标
■ 可量化的成功指标（至少 2 个）

## 风险假设
■ 关键假设（假设不成立时的影响）
□ 已知风险
```

---

## 2. UXArchitectureSpec（体验架构）

```markdown
# 体验架构规格

## 信息架构
■ 页面层级图（树形结构或 ASCII）
■ 导航结构（全局导航 / 面包屑 / 侧边导航）
■ 入口/出口映射

## 用户流程
■ 主路径流程图（Happy Path）
■ 异常路径列表（至少 3 条）

## 交互原则
■ 选用的交互模式（参考 interaction-patterns.md）
■ 反馈策略

## 状态覆盖表
■ 每个核心页面的 6 态覆盖表

| 状态 | 触发条件 | 用户反馈 | 恢复动作 |
|------|---------|---------|---------|
| loading | | | |
| empty | | | |
| error | | | |
| success | | | |
| permission | | | |
| partial | | | |
```

## 3. UIBlueprintSpec（UI 蓝图）

```markdown
# UI 蓝图规格

## 设计系统映射
■ Token 清单（色彩、间距、排版、圆角、阴影）
■ AI 审美偏离度自检结果（参考 anti-ai-aesthetic.md）

## 色彩方案
■ 主色 + 辅色 + 灰阶 + 语义色（含色值）
■ 色彩选择理由（与品牌/业务的关联）
■ WCAG 2.2 AA 对比度验证结果

## 字体方案
■ 字体族选择 + 选择理由
■ 字体尺寸阶梯

## 页面蓝图
■ 关键页面的布局示意（ASCII / 线框描述）
■ 页面原型选择理由（参考 page-archetypes.md）

## 组件树
■ 核心页面的组件树结构
■ 组件职责说明

## 视觉一致性检查
■ 主行动按钮一致性
■ 反馈组件一致性
■ 信息层级一致性
```

## 4. EngineeringHandoffSpec（工程交接）

```markdown
# 工程交接规格

## 技术路由
■ Framework: React / Vue
■ 状态管理策略
■ 数据请求策略

## 模块拆分
■ 组件树（含容器/展示/共享分类）
■ 各模块职责说明

## 数据与错误契约
■ API 契约（接口 / 字段 / 类型）
■ 错误处理策略

## 非功能约束
■ 性能预算（LCP / INP / CLS）
■ 可访问性基线
■ 安全基线
□ 可观测性埋点清单
```

## 6. ReleaseDecision（发布决策）

```markdown
# 发布决策

## 主方案
■ 方案概述

## 备选方案
1. 方案 A — 拒绝原因：
2. 方案 B — 拒绝原因：

## AI 审美偏离度
■ 色彩独特性: ✓/✗

■ 质感真实性: ✓/✗
■ 整体辨识度: ✓/✗
```
